\section{Independent Extensions}
\label{sec:future}

The completed \code{ComputableEuclideanOps} layer supplies Boolean equality,
division with remainder, normalization, units, extended gcd, and a
well-founded measure. It replaces factor-count termination and supports
genuine evaluation over \(\mathbb Z\) and \(\mathbb Q[X]\).

The certificate path now has a pure parser/checker/soundness split, a strict
\code{decide_cbv}-based Lean module generator, and pinned optional FLINT
process and FFI adapters sharing one C computation and raw data schema. Lean
rechecks the expected input, transformation equations, both inverse equations,
and normal-form predicate. Seeded differential regressions and decomposed
generation/check benchmarks are recorded separately from the trusted theorem
path. Truthful operation, pivot, and gcd telemetry remains future
instrumentation rather than an inferred benchmark value.

The 1.0 layer separates intrinsic Smith data from execution traces.
Determinantal ideals identify the rank and associate-class signature under
certified transformations and reindexing. Finite-free presentations fix the
matrix direction \(R^{\mathrm{relations}}\to R^{\mathrm{generators}}\) and
derive the cokernel decomposition. Exact type tests now freeze these
interfaces, and the core release gate checks the pinned container inputs,
local archive metadata, hashes, and citation data.

After the core facade and artifact freeze, rational canonical form, finite-free
integer homology, Kannan--Bachem, modular HNF, LLL, and formal bit-cost models
proceed as independent application or research lines. Each line reuses the
strong core result types and has separate acceptance tests, benchmarks, paper,
and artifact.
